# Otto ↔ 我们的舵机映射参考

## 物理映射（已验证）

| 我们 ID | GPIO | LEDC | Otto 索引 | 关节 | 类型 |
|---------|------|------|-----------|------|------|
| 1 (右脚 RF) | 11 | CH1 | Otto[3] | 右脚踝 | pitch 前后 |
| 2 (右腿 RL) | 12 | CH2 | Otto[1] | 右髋 | yaw 水平旋转 |
| 3 (左腿 LL) | 13 | CH3 | Otto[0] | 左髋 | yaw 水平旋转 |
| 4 (左脚 LF) | 14 | CH4 | Otto[2] | 左脚踝 | pitch 前后 |

**Otto 数组序**: `[LL, RL, LF, RF]` = `[0, 1, 2, 3]`

**我们的 RunOscillation 序**: `[RF, RL, LL, LF]` = `[1, 2, 3, 4]`

```
Otto[0]=LL  →  我们[2]=LL
Otto[1]=RL  →  我们[1]=RL
Otto[2]=LF  →  我们[3]=LF
Otto[3]=RF  →  我们[0]=RF
```

## 角度方向（已验证）

### 腿 (yaw) — 左右对称
| 角度 | RL (右腿) | LL (左腿) |
|------|-----------|-----------|
| 60° | 内旋（脚内收） | 内旋（脚内收） |
| 90° | 中立 | 中立 |
| 120° | 外旋（脚外展） | 外旋（脚外展） |

### 脚 (pitch) — 左右相反
| 角度 | RF (右脚) | LF (左脚) |
|------|-----------|-----------|
| 60° | 脚尖抬起 | 脚尖压下 |
| 90° | 平放 | 平放 |
| 120° | 脚尖压下 | 脚尖抬起 |

**关键**: 左右脚 60°/120° 物理意义相反。

## 相位转换规则

Otto 的 `phase_diff` 需要按以下规则映射到我们的 `phase[]`：

```
Otto phase_diff[LL, RL, LF, RF] → 我们的 phase[RF, RL, LL, LF]

我们的[0] = Otto[3]  (RF)
我们的[1] = Otto[1]  (RL)
我们的[2] = Otto[0]  (LL)
我们的[3] = Otto[2]  (LF)
```

### Walk — 前进方向（已验证）
```
Otto walk:  feet = -90°
我们 walk:  feet = +90°  (即 dir * 90)

我们的 forward = Otto 的 backward（相位符号翻转）
因为 Otto 的正方向定义和我们的舵机安装方向相反
```

### Turn — 原地转（已验证）
```
Turn 必须用 Otto 原始的 -90° 脚相位
+90° 会导致脚在腿中立时抬起而不是压下，失去 pivot 抓地力
```

### Crusaito — 螃蟹步（已验证）
```
RF 相位必须用 Otto 原始的 -60°*dir
不能用我们 walk 的 +60°*dir（会导致反向）
```

## 移植 Otto 动作的步骤

1. 从 Otto 代码复制 `A[]`, `O[]`, `phase_diff[]`（按 Otto 顺序: LL, RL, LF, RF）
2. 按上面的索引映射重排为我们的顺序 (RF, RL, LL, LF)
3. **Walk/Turn 类动作**: 脚相位符号翻转（-90° → +90°）
4. **Crusaito**: RF 相位不翻转，保持 Otto 原始值
5. **其他动作**: 逐个验证方向，不确定时先测试
